# Workflow Mirror — Replit Build Prompt

## Project Overview

Build a working prototype called "Workflow Mirror" — a peer-led AI workflow sharing feature for a fictional B2B SaaS platform called Chico.ai. This is a job interview assignment that must be presentable to a live panel in a 30-minute session.

The prototype must demonstrate a complete core loop: browse peer workflows → view detail → capture a new workflow using Claude AI structuring → publish to feed.

## The Problem Being Solved

Chico.ai helps enterprises adopt AI tools (Claude, ChatGPT, Gemini). Their clients buy 500 AI licences but only ~40 people actively use them. The other 460 have access but don't use AI in their daily work. The current product is a usage analytics dashboard — it shows who isn't using AI but does nothing to fix it.

Research shows the #1 predictor of AI adoption is peer influence (Microsoft Research, 2026 — 88% of heavy AI users cited peers as influential vs 50% of low users). The gap isn't training or access — it's visibility. When AI use stays private, non-adopters have no social proof that it works for their specific role.

Workflow Mirror solves this by letting active AI users share their real workflows, structured automatically by Claude, so non-adopters can browse role-specific examples from peers doing the same job.

## Tech Stack

- **Frontend:** React with TypeScript
- **Styling:** Tailwind CSS + shadcn/ui components (Radix UI primitives)
- **Backend:** Express.js with TypeScript (or Next.js API routes)
- **AI Integration:** Anthropic Claude API for structuring raw workflow input
- **Bundler:** Vite
- **State Management:** React state (useState) — no need for TanStack Query in prototype
- **Data:** JSON seed data for pre-built workflows. New workflows generated via Claude API stored in memory (no database needed for prototype)

## Screen 1 — Workflow Feed (Landing Page)

This is what a non-adopter sees when they open the feature.

### Layout:
- Header bar with "Workflow Mirror" title and Chico.ai branding
- "Share a workflow" button (prominent, top right) — navigates to Screen 2
- Filter bar below header with three dropdowns:
  - **Role:** All Roles, Operations Manager, Account Manager, Sales Analyst, Compliance Officer, HR Business Partner, Business Development Manager
  - **AI Tool:** All Tools, Claude, ChatGPT, Gemini
  - **Category:** All Categories, Reporting, Communication, Analysis, Admin
- Below filters: a grid or list of workflow cards

### Workflow Card Design:
Each card displays:
- **Title** (bold, e.g. "Drafting quarterly board updates")
- **Role tag** (pill/badge, e.g. "Operations Manager")
- **AI tool tag** (pill/badge with tool colour, e.g. "Claude")
- **Time saved** (prominent, e.g. "~2h 15m saved/week")
- **Brief summary** (1-2 sentences max)
- **"This worked for me" count** (small icon + number, e.g. "👍 12")
- Cards are clickable → navigate to Screen 3

### Filtering:
- Filters work client-side on the seeded data
- Selecting a role shows only workflows matching that role
- Filters can be combined (e.g. "Operations Manager" + "Claude")
- "All" option resets each filter

## Screen 2 — Capture (Share a Workflow)

This is where an active AI user records their workflow.

### Layout:
- Header: "Share a workflow"
- Instruction text: "In your own words, describe one thing you did this week using AI that saved you time. Talk normally — we'll structure it for you."
- Large text area (at least 4-5 lines visible)
- Placeholder text in the textarea: "e.g. I had to write the Q1 board update yesterday. Usually takes me 3 hours. I dropped my notes from the last quarterly into Claude, gave it the new numbers, asked it to draft in my voice, then edited for 20 minutes. Done in 45 minutes total."
- Submit button: "Structure my workflow" (purple, prominent)
- Loading state while Claude API processes (spinner or skeleton)

### On Submit — Claude API Call:
Send the raw text to the Claude API with a system prompt that structures it into JSON:

```
System prompt:
"You are a workflow structuring assistant for an enterprise AI adoption platform. The user will describe something they did using an AI tool this week. Extract and structure their description into the following JSON format. Be concise and practical. If information is missing, make a reasonable inference based on the description.

{
  "title": "Short descriptive title for this workflow (max 8 words)",
  "role": "The user's likely job role based on the task described",
  "aiTool": "Claude or ChatGPT or Gemini (whichever was mentioned or most likely)",
  "category": "Reporting or Communication or Analysis or Admin",
  "timeSaved": "Estimated time saved per occurrence (e.g. '~2h 15m')",
  "frequency": "How often this task occurs (e.g. 'Weekly', 'Daily', 'Per project')",
  "summary": "One sentence summary of what they did and the outcome",
  "steps": ["Step 1 description", "Step 2 description", "Step 3 description"],
  "tips": "One practical tip or nuance the user mentioned (or infer one)"
}

Respond with only the JSON object. No markdown, no explanation."
```

### Preview Screen (After Claude Response):
- Display the structured workflow as a card preview
- Each field is shown with an edit button or inline editing
- Fields: Title, Role, AI Tool, Category, Time Saved, Summary, Steps (editable list), Tips
- Two buttons at bottom:
  - "Edit" — allows inline editing of any field
  - "Publish" (purple, prominent) — adds the workflow to the feed and navigates back to Screen 1
- "Back" link to return to feed without publishing

### On Publish:
- Add the new workflow to the in-memory array of workflows
- Navigate back to Screen 1 where the new workflow appears at the top of the feed

## Screen 3 — Workflow Detail

Expanded view when clicking a card from the feed.

### Layout:
- Back button/link to feed
- Full workflow card with all fields:
  - Title (large heading)
  - Role tag + AI Tool tag + Category tag (row of pills)
  - Time saved (prominent)
  - Frequency
  - Summary paragraph
  - Steps listed with numbering (1, 2, 3...)
  - Tips section (highlighted box or callout)
- "This worked for me" button — clicking increments the count (client-side state only, no persistence needed)
- "Try this workflow" button — opens a new tab to the relevant AI tool:
  - Claude → https://claude.ai
  - ChatGPT → https://chatgpt.com
  - Gemini → https://gemini.google.com

## Important Build Notes

1. This is a prototype for a live demo — it needs to work reliably, not be production-ready
2. No authentication needed — skip login/signup entirely
3. No database needed — all data lives in memory and seeded JSON
4. The Claude API call on Screen 2 is the most critical feature — this must work live during the demo
5. Keep the UI clean and professional but don't over-polish — substance over style
6. All three screens should be navigable via client-side routing (React Router or similar)
7. The "This worked for me" button should increment visually but doesn't need persistence
8. New workflows published via Screen 2 should appear in the Screen 1 feed immediately (in-memory state)
9. Error handling on the Claude API call — show a user-friendly message if the API fails, don't crash
10. The whole app should feel like a feature inside a larger SaaS platform, not a standalone website

## What Success Looks Like

When complete, I should be able to:
1. Open the app and see a feed of 6 pre-populated workflow cards
2. Filter by role, AI tool, and category — cards update instantly
3. Click a card to see the full detail view with steps and tips
4. Click "This worked for me" and see the count increment
5. Navigate to "Share a workflow" and type a raw description
6. Hit submit and watch Claude structure it into a card in real time
7. Preview the structured card, edit a field, and publish
8. See the new workflow appear at the top of the feed
